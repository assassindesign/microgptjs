课前准备：我们要造什么？

在开始写代码之前，先用一个生活类比弄清楚目标

GPT 是一种能"预测下一个字"的程序。你输入"今天天气真"，它能预测出"好"。训练时，它不断猜测并被纠正，直到猜得越来越准。

🍎

1

随机数：可控的骰子

线性同余法 —— 用一颗"种子"生成可复现的随机数序列

神经网络在初始化时需要随机数。但普通的随机数每次运行都不同，会导致实验无法复现。代码里自己实现了一个"伪随机数生成器"，这样只要给同一个种子，每次都能产生完全相同的数列。

🎲
类比：魔法骰子

想象你有一颗魔法骰子，规则是：下一次的结果 = 上一次的结果 × 16807，然后除以 2147483647 取余数。只要初始结果（种子）一样，每次摇都会得到同样的序列。这就叫"伪随机数"——看起来随机，其实完全可预测。

代码解析

JSmicrogpt.js 第 8–21 行
var _seed = 42;           // 种子：就像魔法骰子的起始值

function rand() {
    // 线性同余公式：新种子 = 旧种子 × 16807 mod 2147483647
    _seed = (_seed * 16807) % 2147483647;
    // 把结果压缩到 [0, 1) 之间
    return (_seed - 1) / 2147483646;
}

function randn() {
    // Box-Muller 变换：把两个 [0,1) 随机数变成"正态分布"的随机数
    // 正态分布 = 大部分值靠近 0，极端值很少
    var u = rand();
    var v = rand();
    return Math.sqrt(-2 * Math.log(u)) * Math.cos(2 * Math.PI * v);
}

读取数据 & 分词器

把文字变成数字 —— 给每个字发一张"身份证"

计算机只懂数字，不懂文字。所以第一步要把训练数据（英文名字或中文诗词）里的每一个字符，都映射成一个数字编号。这个过程叫分词（Tokenization）。

数据加载流程

1

检查并下载数据

如果本地没有 input.txt，就自动用 curl 从 GitHub 下载英文名字数据集（约 3.2 万个名字）。

2

按行切分成文档

把文件按换行符 \n 切开，每一行（例如 "emma"）就是一条训练数据。

3

随机打乱顺序

用 Fisher-Yates 洗牌算法打乱所有文档，让模型训练时不会只记住顺序。

4

建立词表

统计所有出现过的字符，排好序，每个字符得到一个唯一的数字 ID。还有一个特殊的 BOS（句子开始）符号。

代码解析：建立词表

JS分词器核心代码
// 第一步：收集所有出现过的字符
var chars = {};
for (var i = 0; i < docs.length; i++) {
    var d = docs[i];
    for (var k = 0; k < d.length; k++) {
        chars[d[k]] = true;  // 记录每个字符（去重）
    }
}

// 第二步：排序，保证每次运行词表顺序一致
var uchars = [];
for (var c in chars) { uchars.push(c); }
uchars.sort();

// 第三步：BOS = "句子开始"符号，编号 = 字符数（排在最后）
var BOS = uchars.length;       // 比如英文有26个字母，BOS = 26
var vocabSize = uchars.length + 1;  // 词表大小 = 字符数 + 1（含BOS）

// 字符 ↔ 数字 互转
function charToId(ch) {         // 'a' → 0, 'b' → 1, ...
    for (var i = 0; i < uchars.length; i++) {
        if (uchars[i] === ch) return i;
    }
    return -1;
}
function idToChar(id) { return uchars[id]; }  // 0 → 'a'

3

Value：会记账的数字

自动求导引擎的核心 —— 每个数字都记得自己的"出身"

这是整个程序最重要的概念。普通的数字只知道自己的值，而 Value 对象还额外记住了：① 我是怎么算出来的（children） 和 ② 如果我变大一点，对最终答案有多大影响（grad）。

📒
类比：记账本

普通数字就像钱包里的钱，只知道"有多少"。
Value 就像一本记账本，不仅知道"现在有多少钱"，还记录了"这笔钱从哪来的、怎么算的"。这样以后如果发现亏损（误差），可以顺着记录追回去找出问题。

Value 的数据结构

JSValue 类定义
function Value(data, children, grads) {
    this.data     = data;          // 当前的数值，比如 3.14
    this.grad     = 0;              // 梯度，初始为 0（后面反向传播会填进来）
    this.children = children || [];  // 我是由哪些 Value 算出来的
    this.grads    = grads    || [];  // 每个 child 对我的"贡献比例"（局部导数）
}

5
grad（梯度）
0.0
反向传播后变化
children（父节点）
[a, b]
比如 a + b = 5
grads（局部导数）
[1, 1]
加法中各贡献 = 1
一个 Value 对象的四个字段：值、梯度、来自哪里、各来源的贡献系数

六个基本运算

代码里定义了六个运算，每个运算除了计算数值，还同时记录了"如果结果变化1，每个输入需要变化多少"（这叫局部导数，只用到了小学除法）：

运算数学公式计算值局部导数（grads）小学理解
add(a,b) a + b a.data + b.data [1, 1] a 增加 1，结果也增加 1
mul(a,b) a × b a.data × b.data [b.data, a.data] a 增加 1，结果增加 b 倍
neg(a) -a -a.data [-1] a 增加 1，结果减少 1
pow(a,p) aᵖ a.data^p [p × a.data^(p-1)] 幂函数求导（初中知识）
expv(a) eᵃ Math.exp(a.data) [eᵃ]（自身） 指数函数导数等于自身
relu(a) max(0, a) a>0 ? a : 0 [a>0 ? 1 : 0] 正数通过，负数变0
JS六个基本运算的实现
function add(a, b) {
    // 结果 = a + b，两个 child 的贡献都是 1（加法对每个输入的导数是1）
    return new Value(a.data + b.data, [a, b], [1, 1]);
}

function mul(a, b) {
    // 结果 = a × b，a 的贡献是 b，b 的贡献是 a（乘法交叉）
    return new Value(a.data * b.data, [a, b], [b.data, a.data]);
}

function relu(a) {
    // 大于0就通过，否则输出0。导数：大于0是1，否则是0
    return new Value(
        a.data > 0 ? a.data : 0,
        [a],
        [a.data > 0 ? 1 : 0]
    );
}
💡
为什么 mul 的导数是交叉的？

想象 a=3, b=4，结果=12。
如果 a 增加 1（变成4），结果变成 4×4=16，增加了 4（= b 的值）。
如果 b 增加 1（变成5），结果变成 3×5=15，增加了 3（= a 的值）。
所以 a 对结果的影响系数 = b，b 对结果的影响系数 = a。这就是 [b.data, a.data]。

✏️ 小练习

用代码计算 a = Value(3)，b = Value(4)，c = mul(a, b)。
请问 c.data 是多少？c.grads 是什么？c.children 里有什么？

答案：c.data = 12，c.grads = [4, 3]（b, a的值），c.children = [a, b]

计算图：数字的家谱树

每次运算都生成一棵树，记录"这个结果是怎么来的"

当你把多个 Value 运算串起来，比如 c = add(mul(a, b), a)，就会自动形成一棵计算图（Computational Graph）。这棵树就像家谱——每个节点知道自己的父节点是谁。

🌳
类比：家谱树

计算图就像一棵家谱树。树根是最终结果（损失值），叶子是最原始的输入（模型参数）。每个中间节点都知道"我的父母是谁、他们对我有多大贡献"。反向传播就是从根节点出发，顺着家谱找到每一片叶子，告诉它"因为你的原因，最终答案偏差了多少"。

一个具体例子

假设 a = Value(2)，b = Value(3)，计算 c = a × b + a：

grads: [1, 1]
ab = 6
mul(a, b)
grads: [3, 2]
a = 2
参数叶子
b = 3
参数叶子
a = 2
同一个 a
计算图：c = a×b + a = 2×3 + 2 = 8。注意 a 被用了两次，梯度会累加。

拓扑排序：确定反向传播顺序

反向传播时，必须先处理"输出"节点，再处理"输入"节点。为了找到正确的顺序，代码用了拓扑排序（DFS深度优先搜索）：

JS拓扑排序 —— 建立反向传播顺序
function backward(v) {
    var topo    = [];  // 存排好序的节点（从叶子到根）
    var visited = [];  // 记录已经访问过的节点，避免重复

    function visit(x) {
        if (visited.indexOf(x) !== -1) return;  // 已访问过，跳过
        visited.push(x);
        // 先递归访问所有"子节点"（输入来源）
        for (var i = 0; i < x.children.length; i++) {
            visit(x.children[i]);
        }
        topo.push(x);  // 等所有子节点处理完，再把自己加入
    }

    visit(v);           // 从根节点（最终损失）出发
    v.grad = 1;         // 根节点对自身的梯度 = 1（损失对损失的导数是1）

    // 从根到叶倒序传播梯度
    for (var i = topo.length - 1; i >= 0; i--) {
        var node = topo[i];
        for (var j = 0; j < node.children.length; j++) {
            // 链式法则：child的梯度 += 局部导数 × 当前节点梯度
            node.children[j].grad += node.grads[j] * node.grad;
        }
    }
}
💡
为什么用 += 而不是 =？

因为同一个节点（比如参数 a）可能在计算图里出现多次（上面例子里 a 被用了两次），每次出现都会贡献一部分梯度，所以要累加而不是覆盖。

5

反向传播：顺藤摸瓜找错误

链式法则 —— 把最终误差逆向分摊到每一个参数上

反向传播是神经网络学习的核心机制。它回答一个问题："如果最终的损失（错误）变化了1，每个参数需要负多少责任？"

🕵️
类比：顺藤摸瓜找锅

工厂生产了一批不合格产品（损失大）。老板要追责：先找到最后一道工序的责任，再往上追每一个上游工序的责任，一直追到最开始的原材料（模型参数）。每一步责任的传递，就是"局部导数 × 上游梯度"。

链式法则（用小学乘法理解）

假设有三个环节：参数 w → 中间值 z → 损失 L

w = 2.0
参数（叶节点）
grad = ?
→
z = w × 3
dz/dw = 3
grad = 0.6
→
L = z²
dL/dz = 2z = 12
grad = 1（根节点）
链式法则：dL/dw = (dL/dz) × (dz/dw) = 1 × 12 × 3 = 36
意思是：w 增加 1，L 会增加约 36

完整的反向传播过程（用上面 c = a×b + a 的例子）

已知：a=2, b=3, c=8，现在运行 backward(c)：

1

初始化根节点

c.grad = 1（损失对自身的导数 = 1）

2

处理 add 节点（c = ab + a）

add 的 grads = [1, 1]
→ ab.grad += 1 × c.grad = 1 × 1 = 1
→ a.grad += 1 × c.grad = 1 × 1 = 1（第一次累加）

3

处理 mul 节点（ab = a × b）

mul 的 grads = [b.data, a.data] = [3, 2]
→ a.grad += 3 × ab.grad = 3 × 1 = 3（第二次累加，a.grad 现在 = 1+3 = 4）
→ b.grad += 2 × ab.grad = 2 × 1 = 2

4

最终结果

a.grad = 4（如果 a 增加1，c 增加约 4）
b.grad = 2（如果 b 增加1，c 增加约 2）
优化时：把 a 减小一点点，c（损失）就会减小约 4 倍。

✅
验证：手算检查

c = a×b + a = a(b+1)。对 a 求导 = b+1 = 3+1 = 4 ✓
对 b 求导 = a = 2 ✓
反向传播的结果和手算完全吻合！

✏️ 小练习

function matrix(rows, cols) {
    var m = [];
    for (var i = 0; i < rows; i++) {
        var r = [];
        for (var j = 0; j < cols; j++) {
            r.push(new Value(randn() * 0.08));
            // randn() * 0.08 = 小随机数（接近0但不完全是0）
            // 初始化太大会"爆炸"，太小会"消失"，0.08是经验值
        }
        m.push(r);
    }
    return m;
}

⚠️
为什么初始值要乘以 0.08？

如果初始值太大（比如随机到10），经过多层运算会"爆炸"（越来越大，模型无法收敛）；如果全是0，梯度也是0，无法学习。randn() * 0.08 让初始值在 [-0.24, 0.24] 范围内，保证训练稳定开始。

基础算子：线性层、Softmax、归一化

GPT 模型的三块"积木"

① linear（线性变换）—— 矩阵乘法

这是神经网络最基本的操作：把一个向量（数字列表）通过一个权重矩阵"变换"成另一个向量。本质上就是很多个"加权求和"。

⚖️
类比：投票计分

有5个评委（输入向量），每个评委的打分要乘以不同的权重，再全部加起来，得到最终分数（输出的一个维度）。有多少个"裁判席位"（输出维度），就做多少次这样的加权求和。

JSlinear 函数
function linear(x, w) {
    // x: 输入向量 [x0, x1, ..., xN]
    // w: 权重矩阵 [outputDim 行, inputDim 列]
    // 输出: 每一行的权重与输入的点积
    var out = [];
    for (var i = 0; i < w.length; i++) {   // 遍历输出的每个维度
        var sum = new Value(0);
        for (var j = 0; j < x.length; j++) {  // 对输入每个维度求加权和
            sum = add(sum, mul(w[i][j], x[j]));  // sum += w[i][j] * x[j]
        }
        out.push(sum);
    }
    return out;
}

linear 就是矩阵乘法：每行权重与输入做点积，得到输出的一个维度

② softmax —— 把分数变成概率

模型最后输出的是每个词的"得分"，但我们需要的是概率（所有词的概率加起来 = 1）。Softmax 就是做这个转换的。

🎯
类比：投票比例

3个候选词得分是 [2, 1, 0]。Softmax 先把每个分数做 e^x（指数），得到 [e², e¹, e⁰] ≈ [7.39, 2.72, 1.00]，然后除以总和 11.11，得到概率 [66%, 24%, 9%]。分数越高，概率越大。

JSsoftmax 函数（含数值稳定性处理）
function softmax(xs) {
    // 技巧：先减去最大值，防止 e^很大的数溢出（数值稳定性）
    var max = xs[0].data;
    for (var i = 1; i < xs.length; i++) {
        if (xs[i].data > max) max = xs[i].data;
    }

    var exps = [];
    var sum = new Value(0);
    for (var i = 0; i < xs.length; i++) {
        // e^(x - max)：减去最大值后再求指数，结果相同但不会溢出
        var e = expv(sub(xs[i], new Value(max)));
        exps.push(e);
        sum = add(sum, e);
    }

    var probs = [];
    for (var i = 0; i < exps.length; i++) {
        // 每个 e^x 除以总和 = 概率
        probs.push(mul(exps[i], pow(sum, -1)));
    }
    return probs;
}

③ rmsnorm —— 让数字保持"规范"

在神经网络深处，数字可能越来越大或越来越小，导致训练不稳定。RMSNorm（均方根归一化）把所有维度的值"缩放"到合理范围。

📏
类比：调整音量

录音时有些声音太大、有些太小。归一化就像自动调音量：先测量整段声音的"平均能量"，然后把每个声音除以这个能量值，让整体音量统一。

JSrmsnorm 函数
function rmsnorm(x) {
    // 第一步：计算所有值的平方和 ÷ 个数（均方值）
    var sum = new Value(0);
    for (var i = 0; i < x.length; i++) {
        sum = add(sum, mul(x[i], x[i]));  // sum += x[i]²
    }
    var mean  = mul(sum, new Value(1 / x.length)); // 均方值
    // 第二步：scale = 1 / sqrt(均方值 + 极小数)，防止除以0
    var scale = pow(add(mean, new Value(1e-5)), -0.5);

    var out = [];
    for (var i = 0; i < x.length; i++) {
        out.push(mul(x[i], scale));  // 每个值乘以缩放因子
    }
    return out;
}
输入 x = [4, -2, 4]
均方 = (16+4+16)/3 ≈ 12
RMS = √12 ≈ 3.46
→
输出 = x / RMS
[4/3.46, -2/3.46, 4/3.46]
≈ [1.15, -0.58, 1.15]
RMSNorm 把值缩放到更小的范围，让后续计算更稳定

注意力机制：谁跟谁有关系？

多头自注意力 —— GPT 最核心的创新，让词语学会"关注"彼此

注意力机制是 GPT 最关键的部分。它让模型在处理每个词时，能"回头看"之前出现过的所有词，并判断哪些词对当前位置最重要。

🔍
类比：考试找答案

老师问"那个小明去了哪里？"你要在文章里找"小明"这个词附近的信息。注意力机制就是这样：当前词（"哪里"）像一个"问题"（Query），文章里的每个词是"线索"（Key），答案信息（Value）根据相关度加权提取。

三个核心概念：Q、K、V

❓

Query（问题）

当前词想问的问题：
"我需要什么信息？"

🔑

Key（关键词）

每个历史词的"标签"：
"我能提供什么信息？"

📦

Value（内容）

每个历史词的实际内容：
"我的具体信息是什么？"

📊

Score（相关度）

Q 和 K 的点积，越大代表越相关，用 Softmax 转成权重

注意力计算步骤

1

把输入变成 Q、K、V

把当前位置的嵌入向量 x，分别通过三个不同的权重矩阵（attn_wq/wk/wv），变换成三个向量 q、k、v。

2

计算相关度分数

把当前的 q 和每个历史位置的 k 做点积，再除以 √headDim（防止值太大导致 softmax 梯度消失）。

score(t) = (q · k_t) / √headDim
3

Softmax 转成注意力权重

把所有历史位置的分数做 Softmax，得到概率分布（权重之和 = 1）。分数越高，该历史位置被"关注"越多。

4

加权求和 Values

用注意力权重对所有历史位置的 v 做加权平均，得到这个"头"的输出。

attn_out = Σ weight(t) × v(t)

代码解析：多头注意力

JS注意力机制核心代码（gpt 函数内）
// 计算 Q K V（通过线性变换）
var q = linear(x, state.attn_wq);  // 当前位置的 Query（问题）
var k = linear(x, state.attn_wk);  // 当前位置的 Key（标签）
var v = linear(x, state.attn_wv);  // 当前位置的 Value（内容）

keys.push(k);    // 把当前的 k 加入历史 Key 列表
values.push(v); // 把当前的 v 加入历史 Value 列表

// 多头注意力：nHead 个头并行计算
for (var h = 0; h < nHead; h++) {
    var hs = h * headDim;  // 每个头负责的维度起始位置

    // 计算当前头与所有历史位置的相关度分数
    var logits = [];
    for (var t = 0; t < keys.length; t++) {
        var score = new Value(0);
        for (j = 0; j < headDim; j++) {
            // q 与 k 的点积（只在本头负责的维度）
            score = add(score, mul(q[hs+j], keys[t][hs+j]));
        }
        // 除以 √headDim 防止值太大（缩放点积）
        score = mul(score, new Value(1 / Math.sqrt(headDim)));
        logits.push(score);
    }

    // 用 softmax 把分数变成注意力权重（概率）
    var weights = softmax(logits);

    // 用权重对 Value 加权求和
    for (var j = 0; j < headDim; j++) {
        var sum = new Value(0);
        for (var t = 0; t < values.length; t++) {
            sum = add(sum, mul(weights[t], values[t][hs+j]));
        }
        attn_out[hs+j] = sum;  // 存入对应维度
    }
}
💡
为什么要"多头"？

4个头同时工作，每个头只关注向量的一部分（4维），专注于不同类型的关系：有的头学语法关系，有的头学语义关系，有的头学位置关系。最后把4个头的结果拼在一起，获得更丰富的表达。

✏️ 小练习：理解注意力权重

假设处理句子 "the cat sat"，当处理 "sat" 时：

  • q（sat的问题）与 k（the的标签）点积 = 0.1
  • q（sat的问题）与 k（cat的标签）点积 = 2.5
  • q（sat的问题）与 k（sat的标签）点积 = 1.0

经过 softmax 后，"cat" 的注意力权重最大，说明 "sat" 这个词最关注 "cat"——这符合语法直觉（是猫在坐）。

MLP 前馈层：每个位置独立思考

注意力之后的"深度思考"模块 —— 扩展、激活、压缩

注意力机制负责"信息汇聚"（把所有历史位置的信息整合到当前位置），而 MLP（多层感知机）负责"深度加工"——对汇聚后的信息做非线性变换，提取更复杂的特征。

🧪
类比：过滤器

注意力就像把很多原材料混合在一起，而 MLP 就像化学实验：先把混合物放进一个大容器扩展（16→64），在里面发生化学反应（ReLU 激活：只保留"正反应"），然后再提炼浓缩回去（64→16）。这个过程可以提取出更深层的特征。

MLP 的三步流程

JSMLP 前馈层代码
// 注意力之后，残差连接保存原始信息
var proj = linear(attn_out, state.attn_wo);  // 注意力输出投影
x_res = x;  // 保存当前 x 作为残差（"跳过连接"的原始值）
for (var i = 0; i < nEmb; i++) {
    x[i] = add(proj[i], x_res[i]);  // x = 注意力输出 + 残差
}

// MLP 开始
x_res = x;                        // 再次保存残差
x = rmsnorm(x);                   // 第一步：归一化

x = linear(x, state.mlp_fc1);    // 第二步：扩展 16 → 64（nEmb → 4×nEmb）

for (var i = 0; i < x.length; i++) {
    x[i] = relu(x[i]);             // 第三步：ReLU 激活（负值清零）
}

x = linear(x, state.mlp_fc2);    // 第四步：压缩 64 → 16（4×nEmb → nEmb）

for (var i = 0; i < nEmb; i++) {
    x[i] = add(x[i], x_res[i]);    // 第五步：加上残差连接
}

⚡
什么是残差连接？

残差连接就是"抄近路"：把处理前的 x 直接加到处理后的 x 上。这样即使中间层出了问题（梯度为0），原始信息也能通过"近路"直接传到后面，防止深层网络训练困难。就像考试允许你把草稿纸带进去，万一脑子卡住了，直接看草稿。

GPT 完整前向传播

把词嵌入、注意力、MLP 串起来，输出下一个词的概率

前面学的所有模块，现在串联起来就是 GPT 的完整前向传播过程。函数 gpt(tokenId, posId, keys, values) 每次处理一个 token（字符），返回对下一个字符的预测分数。

gpt 函数完整流程

从 wte 取出当前词的向量，从 wpe 取出当前位置的向量，两个向量对应位置相加，得到"这个词在这个位置"的综合表示 x。

x[i] = wte[tokenId][i] + wpe[posId][i]

第一次 RMSNorm

对 x 做归一化，保持数值稳定。

x_res = x，留作后面残差连接用。

Adam 优化器：聪明的学习方式

比简单梯度下降更聪明 —— 记住历史，动态调整步长

反向传播得到每个参数的梯度（知道了应该往哪个方向调整），接下来优化器负责决定"调整多少"。Adam 是目前最常用的优化器之一。

🏃
类比：聪明的球滚下山

普通梯度下降像一个盲目的球：每次只看眼前坡度，步子固定，可能在谷底反复震荡。
Adam 像一个聪明的滑雪者：① 有惯性（动量，m）—— 记住之前的方向继续滑；② 会根据地形自适应（v）—— 坡度起伏大的地方步子小，平坦的地方步子大。

Adam 的两个"记忆"

变量名称记住什么类比
m[i]一阶矩（动量）梯度的"移动平均"之前走的方向的记忆
v[i]二阶矩（自适应）梯度平方的"移动平均"这个方向历史上有多"抖动"

Adam 更新公式（逐行解析）

JSAdam 优化器更新步骤
var lr    = 0.01;    // 学习率：每步最多走多远
var beta1 = 0.85;   // 动量系数：0.85 = 记住85%的历史方向
var beta2 = 0.99;   // 自适应系数：0.99 = 记住99%的历史抖动
var eps   = 1e-8;   // 极小数，防止除以0

// 学习率随训练进度线性衰减（开始快，后期慢）
var lr_t = lr * (1 - step / steps);

for (var i = 0; i < params.length; i++) {
    // 第一步：更新动量 m（梯度的指数移动平均）
    m[i] = beta1 * m[i] + (1 - beta1) * params[i].grad;
    // = 85% × 历史方向 + 15% × 当前梯度

    // 第二步：更新自适应 v（梯度平方的指数移动平均）
    v[i] = beta2 * v[i] + (1 - beta2) * params[i].grad * params[i].grad;
    // = 99% × 历史抖动 + 1% × 当前梯度²

    // 第三步：偏差修正（训练初期 m/v 接近0，需要放大）
    var mh = m[i] / (1 - Math.pow(beta1, step+1));  // 校正后的 m
    var vh = v[i] / (1 - Math.pow(beta2, step+1));  // 校正后的 v

    // 第四步：更新参数（沿梯度反方向走一小步）
    params[i].data -= lr_t * mh / (Math.sqrt(vh) + eps);
    // lr_t × mh/√vh：动量方向 ÷ 抖动程度 = 自适应步长

    // 清零梯度（下次反向传播重新计算）
    params[i].grad = 0;
}
Adam 更新的直觉理解（以一个参数为例）：
g
当前梯度 = +0.5（需要减小参数）
m
动量 = 0.85×(旧m) + 0.15×0.5，平滑方向
v
自适应 = 0.99×(旧v) + 0.01×0.25，记录抖动
Δ
参数更新 = -lr × m/√v，抖动大则步子小
Adam 的核心思想：用历史信息动态调整每个参数的学习步长

✏️ 小练习：手算一步 Adam

给定：lr=0.01, beta1=0.9, beta2=0.99, step=0
初始 m=0, v=0，当前梯度 g=0.4

  1. m = 0.9×0 + 0.1×0.4 = 0.04
  2. v = 0.99×0 + 0.01×0.16 = 0.0016
  3. mh = 0.04 / (1 - 0.9¹) = 0.04/0.1 = 0.4
  4. vh = 0.0016 / (1 - 0.99¹) = 0.0016/0.01 = 0.16
  5. Δ = -0.01 × 0.4 / (√0.16 + 1e-8) ≈ -0.01 × 0.4/0.4 = -0.01

偏差修正的作用：把实际 m=0.04 "放大"成 0.4，因为训练刚开始动量还没积累够。

训练循环：反复练习直到掌握

把所有模块串起来，让模型不断从错误中学习

训练循环就是把前向传播、损失计算、反向传播、参数更新这四步不断重复，每次重复叫一个"step（步骤）"。

损失函数：交叉熵

模型的"错误"用交叉熵损失来衡量。公式很简单：-log(正确答案的概率)。概率越高（模型越自信且正确），损失越小；概率越低，损失越大。

正确答案概率 = 0.9
损失 = -log(0.9) ≈ 0.1
模型很自信，损失小
正确答案概率 = 0.1
损失 = -log(0.1) ≈ 2.3
模型不自信，损失大
交叉熵损失：概率越接近1，损失越接近0；概率越低，损失越大

训练循环代码解析

JS训练循环完整代码
var steps = 1000;          // 总训练步数
var startStep = loadCheckpoint(); // 从 checkpoint 继续或从0开始

for (var step = startStep; step < steps; step++) {

    // ① 选一条训练数据（循环使用所有文档）
    var doc = docs[step % docs.length];

    // ② 把文档变成 token 序列（加上首尾的 BOS）
    var tokens = [BOS];
    for (var i = 0; i < doc.length; i++) {
        tokens.push(charToId(doc[i]));
    }
    tokens.push(BOS);  // 结尾 BOS 作为"停止"信号

    // ③ 前向传播：逐个 token 预测"下一个"，累计损失
    var keys = [], values = [];
    var loss = new Value(0);

    for (var i = 0; i < tokens.length - 1 && i < blockSize; i++) {
        var logits = gpt(tokens[i], i, keys, values);
        var probs  = softmax(logits);
        // 交叉熵：加上 -log(正确答案的概率)
        loss = add(loss, neg(logv(probs[tokens[i+1]])));
    }

    // ④ 反向传播：计算所有参数的梯度
    backward(loss);

    // ⑤ Adam 优化：用梯度更新参数
    var lr_t = lr * (1 - step / steps);
    for (var i = 0; i < params.length; i++) {
        m[i] = beta1*m[i] + (1-beta1)*params[i].grad;
        v[i] = beta2*v[i] + (1-beta2)*params[i].grad*params[i].grad;
        var mh = m[i] / (1-Math.pow(beta1,step+1));
        var vh = v[i] / (1-Math.pow(beta2,step+1));
        params[i].data -= lr_t * mh / (Math.sqrt(vh) + eps);
        params[i].grad = 0;  // 梯度清零
    }

    // ⑥ 每20步打印一次损失
    if (step % 20 === 0) {
        console.log('step', step, 'loss', loss.data.toFixed(4));
    }
}
📉
训练成功的标志

如果训练正常，损失（loss）应该从初始值（比如 3.5）随着步数增加而逐渐减小（比如降到 1.5 以下）。损失越小，模型对数据的理解越准确。

模型保存与断点续训

把训练结果保存到文件，随时可以继续

训练一个模型需要很长时间。如果中途断电或关闭程序，之前的训练成果就丢失了。saveCheckpoint 和 loadCheckpoint 解决这个问题：每隔20步保存一次，下次直接从断点继续。

保存的内容

字段内容为什么要保存
wte, wpe, 等权重所有参数矩阵的数值这就是"学到的知识"
_m, _vAdam 的 m 和 v 数组不保存的话，续训时动量从0重来
_step当前训练步数告诉程序从哪一步继续
_seed随机数种子保证续训时数据顺序一致
_uchars词表字符列表确保新运行时词表和保存时完全一致
JScheckpoint 核心逻辑
function saveCheckpoint(nextStep) {
    var saved = {};
    saved._step  = nextStep;    // 记录到了第几步
    saved._seed  = _seed;       // 保存随机数状态
    saved._uchars = uchars;     // 保存词表

    // 保存 Adam 状态
    saved._m = []; saved._v = [];
    for (var i = 0; i < m.length; i++) {
        saved._m.push(m[i]);
        saved._v.push(v[i]);
    }

    // 把所有参数矩阵的 .data 值提取出来，存为普通数字
    for (var s = 0; s < stateKeys.length; s++) {
        saved[stateKeys[s]] = matrixToData(state[stateKeys[s]]);
    }

    // 写入 JSON 文件
    fs.writeFileSync(checkpointPath, JSON.stringify(saved));
}

function loadCheckpoint() {
    if (!fs.existsSync(checkpointPath)) return 0;  // 没有文件就从0开始

    var data = JSON.parse(fs.readFileSync(checkpointPath));
    checkChars(data._uchars);  // 确认词表匹配

    // 恢复参数、Adam状态、步数、种子
    for (var s = 0; s < stateKeys.length; s++) {
        loadMatrixData(stateKeys[s], data[stateKeys[s]]);
    }
    _seed = data._seed;
    return data._step;  // 返回训练步数，从这里继续
}
⚠️
词表不匹配时会拒绝加载

如果你换了 input.txt（比如从英文名字换成中文诗词），词表会变化，旧 checkpoint 里的参数对应的词就错了。代码里的 checkChars 会检测到这种情况并报错，防止你用错误的参数继续训练。

推理生成：让模型"说话"

用训练好的模型生成新文字 —— 自回归采样

训练完成后，给模型一个 BOS（开始信号），它就能一个字一个字地生成内容。这个过程叫自回归生成（Autoregressive Generation）：每次生成的字，会作为下一步的输入。

✍️
类比：接龙游戏

你说"今"，模型说"天"；然后你把"今天"给模型，它说"真"；再把"今天真"给它，它说"好"……就这样一直接龙，直到生成结束符号（BOS）。

Temperature（温度）：控制创意程度

推理时有个参数叫 temperature（温度），控制生成的"创意程度"：

温度值效果类比
低（如 0.2）总选概率最高的词，保守且重复每次都点同一道菜
中（如 0.5）平衡创意和合理性偶尔尝试新菜
高（如 1.5）随机性很强，可能出现意外随机指着菜单点
JS推理生成完整代码
for (var s = 0; s < 10; s++) {   // 生成10条样本
    var keys = [], values = [];
    var token = BOS;       // 从 BOS 开始
    var out = [];

    for (var i = 0; i < blockSize; i++) {
        // 前向传播，得到每个词的得分
        var logits = gpt(token, i, keys, values);

        // 用 temperature 缩放 logits（除以温度）
        var temperature = 0.5;
        var scaledLogits = [];
        for (var k = 0; k < logits.length; k++) {
            scaledLogits.push(mul(logits[k], new Value(1/temperature)));
        }
        var probs = softmax(scaledLogits);  // 变成概率

        // 按概率随机采样下一个 token（累积分布采样）
        var r   = rand();   // 生成 [0,1) 的随机数
        var acc = 0;
        var next = BOS;    // 默认选 BOS（结束符）
        for (var j = 0; j < probs.length; j++) {
            acc += probs[j].data;
            if (r < acc) { next = j; break; }
        }

        if (next === BOS) break;   // 遇到结束符，停止生成
        out.push(idToChar(next)); // 把 ID 转回字符
        token = next;              // 把当前输出作为下一步输入
    }

    console.log(out.join(''));  // 打印生成的名字
}
🎲
累积分布采样（如何按概率随机选词）

假设词表有3个字，概率是 [0.6, 0.3, 0.1]。累积分布是 [0.6, 0.9, 1.0]。
生成随机数 r=0.75：遍历到第2个字时，acc=0.9 ≥ 0.75，选第2个字。
这样高概率的字被选中的可能性更大，但低概率的字也有机会被选中（保持多样性）。

常见错误与排查

🔥
loss 变成 NaN（非数字）

原因：参数初始值太大，或学习率太高，导致数值"爆炸"。
排查：检查 randn 有没有乘以 0.08；检查 lr 是否正确；检查 rmsnorm 里有没有加 1e-5。

📈
loss 不下降

原因：梯度清零位置不对，或 backward 逻辑有 bug。
排查：确认每步训练后 params[i].grad 都被设为 0；用简单例子（如单个乘法）验证 backward 结果。

🔁
生成的文字全是同一个字

原因：temperature 太低（趋近于0），每次都选最高概率的字。
排查：把 temperature 调高（如 0.8 或 1.0）；确认 rand() 每次返回值不同。
